# Project Structure

This document outlines the directory structure of the `meowsenger` project to help AI agents and developers navigate the codebase efficiently.

## Root Directory (`meowsenger/`)

| Directory | Description |
| :--- | :--- |
| `app/` | Contains the Next.js App Router structure, including pages, layouts, and API routes. |
| `components/` | Reusable React components used throughout the application. |
| `context/` | React Context Providers for global state management (Auth, Socket). |
| `hooks/` | Custom React hooks for encapsulating logic. |
| `lib/` | Core libraries, configuration, and singletons. |
| `pages/` | (Legacy/Optional) Next.js Pages Router directory. Most logic is in `app/`. |
| `prisma/` | Database schema (`schema.prisma`) and migration history. |
| `public/` | Static assets like images, fonts, and icons. |
| `utils/` | Utility functions and helpers. |

## Key Subdirectories

### `app/`
*   `api/`: Backend API routes (REST endpoints).
    *   `auth/`: Login and Signup handlers.
    *   `users/`: User search and profile endpoints.
    *   `chats/`: Chat creation and management.
*   `(auth)/`: Route group for authentication pages (`login`, `signup`).
*   `c/`: Route for channels (e.g., `/c/[slug]`).
*   `chat/`: Main chat interface routes.
*   `join/`: Invite link handling.

### `lib/`
*   `prisma.ts`: The singleton instance of the Prisma Client.
*   `site-config.ts`: Global configuration constants (site name, description).

### `utils/`
*   `crypto.ts`: **Critical.** Contains all cryptographic functions for key generation, encryption, and decryption.
*   `inviteActions.ts`: Server actions or utilities related to handling invite codes.

### `prisma/`
*   `schema.prisma`: The source of truth for the database schema.
*   `migrations/`: SQL migration files generated by Prisma.
